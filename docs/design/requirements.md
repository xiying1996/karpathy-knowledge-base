# Karpathy 式个人知识库 — 完整需求文档

## 1. 项目概述

### 1.1 背景

Karpathy（Andrej Karpathy）是深度学习和 NLP 领域最具影响力的研究者之一，曾任 Tesla AI 总监、OpenAI 创始成员。他的个人知识管理系统（Second Brain）是其持续产出的重要支撑。

Karpathy 的知识库方法论具有以下显著特征：
- **工具极简主义**：只用 Obsidian（本地 Markdown）+ 自建工具
- **AI 优先**：大量使用 AI 进行笔记索引、检索和问答
- **强调可复现性**：所有代码、模型、实验都版本化管理
- **工作流驱动**：笔记不是目的，服务于研究和工程工作流

### 1.2 目标

构建一个**中文个人知识管理系统**，以 Karpathy 方法论为蓝本，融合：
- Obsidian 作为笔记载体
- AI 驱动的笔记索引和 RAG 检索
- 双向链接 + 知识图谱可视化
- 支持中文知识沉淀和 AI 问答

## 2. 核心功能定义

### 2.1 笔记管理（Foundation Layer）

#### 2.1.1 基础笔记操作
| 功能 | 描述 | 优先级 |
|------|------|--------|
| 创建笔记 | 支持 Markdown 编辑，支持双链语法 `[[笔记名]]` | P0 |
| 文件夹组织 | 树形目录结构，支持多级嵌套 | P0 |
| 标签系统 | 多维度标签（#tag），支持嵌套标签 | P0 |
| 快速搜索 | 标题/内容全文检索，支持正则 | P0 |
| 模板系统 | 预定义笔记模板（每日笔记、项目笔记、概念卡） | P1 |
| 嵌入块 | `![[笔记名]]` 嵌入其他笔记片段 | P1 |
| MOC（Map of Content）| 主题索引页，聚合相关笔记 | P1 |

#### 2.1.2 特殊笔记类型
```
Daily Notes（每日笔记）
├── 自动按日期归档（YYYY-MM-DD）
├── 包含当日输入来源（阅读/论文/视频/对话）
├── 与项目笔记的双向链接
└── 每日复盘模板

Project Notes（项目笔记）
├── 独立文件夹
├── 包含 README（项目概述）
├── 包含 experiment log
└── 关联相关论文/资料

Concept Cards（概念卡）
├── 一个概念 = 一篇笔记
├── 标准化格式：定义/例子/关联/代码
└── 用于构建个人百科

Paper Notes（论文笔记）
├── 论文元信息（标题/作者/年份/链接）
├── 核心贡献摘要
├── 关键技术细节
├── 代码实现笔记
└── 与其他论文的关联
```

### 2.2 AI 检索层（RAG Implementation）

#### 2.2.1 RAG 架构

```
User Query
    ↓
Query Processing（意图识别 + 关键词提取）
    ↓
Vector Search（Embedding 模型）
    ↓
Top-K 相关笔记片段
    ↓
LLM Generation（带上下文）
    ↓
Answer + Source Attribution
```

#### 2.2.2 RAG 核心功能
| 功能 | 描述 | 技术实现 |
|------|------|----------|
| 语义搜索 | 自然语言查询，返回最相关笔记 | Embedding + 向量数据库 |
| 问答模式 | 基于笔记库的 RAG 问答 | LLM + 上下文注入 |
| 笔记推荐 | 读取当前笔记时推荐相关笔记 | 图谱 + 相似度 |
| 自动摘要 | 长笔记自动生成摘要 | LLM summarization |
| 标签建议 | AI 建议新笔记的标签 | LLM classification |
| 知识缺口检测 | 识别知识体系中的空白 | LLM analysis |

#### 2.2.3 Embedding 策略
- **中文优化**：使用中文 Embedding 模型（如 BGE、Text2Vec）
- **分块策略**：按段落分块（512 tokens），重叠 50 tokens
- **元数据保留**：笔记标题、标签、创建时间、来源 URL

### 2.3 双向链接和知识图谱

#### 2.3.1 链接类型
```markdown
[[笔记名]]              → 双向链接
[[笔记名#章节]]         → 块引用
![[笔记名]]             → 嵌入内容
[[笔记名|显示文本]]      → 别名链接
```

#### 2.3.2 知识图谱功能
| 功能 | 描述 |
|------|------|
| 本地图谱视图 | Obsidian 内置 Graph View |
| 全局图谱 | Web 界面展示完整知识网络 |
| 路径发现 | 找到两个概念之间的关联路径 |
| 聚类分析 | 自动发现知识密集区域和空白区域 |
| 链接质量评估 | 识别孤立笔记（无出链/入链） |

#### 2.3.3 图谱数据模型
```json
{
  "nodes": [
    {"id": "note_001", "title": "注意力机制", "type": "concept", "tags": ["深度学习", "NLP"]}
  ],
  "edges": [
    {"source": "note_001", "target": "note_002", "type": "relates_to", "weight": 0.8}
  ]
}
```

## 3. 用户旅程分析

### 3.1 日常笔记流程（Daily Workflow）

```
┌─────────────────────────────────────────────────────────┐
│  Morning: 日计划                                         │
│  1. 打开今日笔记模板（自动按日期创建）                        │
│  2. 列出今日待办                                          │
│  3. 标记输入来源（今天要读什么）                            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Working Session: 学习和研究                              │
│  1. 阅读论文/文章/视频                                    │
│  2. 在笔记中记录关键发现                                   │
│  3. 使用模板生成结构化笔记                                 │
│  4. 创建双向链接到已有知识                                 │
│  5. 打标签分类                                            │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Evening: 复盘和沉淀                                       │
│  1. 更新 MOC（Map of Content）                            │
│  2. AI 提问：今天学到了什么？                              │
│  3. 标记未完成的思考到明日待办                             │
└─────────────────────────────────────────────────────────┘
```

### 3.2 AI 问答交互流程（Research Workflow）

```
┌─────────────────────────────────────────────────────────┐
│  Query: "Transformer 的注意力机制是如何工作的？"             │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 1: 意图识别                                         │
│  - 检测是概念查询/项目查询/历史记录                         │
│  - 提取关键词：Transformer, 注意力机制                     │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 2: 检索阶段                                         │
│  - 向量搜索：top-5 相关笔记片段                            │
│  - 关键词搜索：补充精确匹配结果                             │
│  - 优先检索高引用/高权重的笔记                             │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Step 3: 生成阶段                                         │
│  - LLM 结合上下文生成答案                                 │
│  - 标注来源笔记（可点击跳转）                              │
│  - 生成追问建议                                           │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Output                                                  │
│  "根据你的笔记《注意力机制》和《Transformer架构》：\n"        │
│  "1. Self-Attention 计算过程...\n"                       │
│  "2. 与 RNN 的对比...\n"                                  │
│  "来源：[[注意力机制]], [[Transformer架构]]"              │
│  "\n💡 追问建议："                                       │
│  "• Multi-head attention 的作用是什么？"                  │
│  "• 如何用 PyTorch 实现注意力？"                          │
└─────────────────────────────────────────────────────────┘
```

### 3.3 知识沉淀和回顾流程（Review Workflow）

```
┌─────────────────────────────────────────────────────────┐
│  Weekly Review（每周日）                                  │
│  - 汇总本周笔记                                            │
│  - AI 生成知识摘要                                         │
│  - 识别需要深化的概念                                       │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Monthly Deep Dive（每月）                                │
│  - 聚焦某个主题进行系统梳理                                 │
│  - 更新相关 MOC                                           │
│  - 整理成可分享的文档                                      │
└─────────────────────────────────────────────────────────┘
                          ↓
┌─────────────────────────────────────────────────────────┐
│  Spaced Repetition（间隔复习）                            │
│  - AI 定期推送需要复习的笔记                               │
│  - 基于遗忘曲线安排复习时间                                 │
│  - 通过小测验检验掌握程度                                   │
└─────────────────────────────────────────────────────────┘
```

## 4. 数据流架构

### 4.1 信息流转

```
[信息输入]                    [知识处理]                   [输出/应用]
────────────────────────────────────────────────────────────────────
  论文/博客       ──┐
  视频/课程       ──┤     Obsidian Vault                AI 问答
  工作会议       ──┼──▶  (Markdown Files)  ──────▶      搜索发现
  个人思考       ──┤          │                         图谱可视化
  代码实验       ──┘          ▼                         写作辅助
                   Embedding  向量数据库
                   Generator LLM API
```

### 4.2 核心数据实体

```python
# 笔记 (Note)
Note {
    id: str (UUID)
    title: str
    content: str (Markdown)
    tags: list[str]
    created_at: datetime
    updated_at: datetime
    links: list[str]  # outgoing links
    backlinks: list[str]  # incoming links (computed)
    source_url: str | null
    note_type: Literal["daily", "project", "concept", "paper", "general"]
}

# 链接关系 (Link)
Link {
    source_id: str
    target_id: str
    link_type: str  # "relates_to", "derives_from", "contradicts", etc.
    context: str  # surrounding text
}

# 嵌入向量 (Embedding)
Embedding {
    note_id: str
    chunk_index: int
    content: str
    vector: list[float]  # 768-dim or 1536-dim
    metadata: dict
}
```

## 5. 非功能性需求

### 5.1 性能要求
- 搜索响应时间 < 500ms（单次查询）
- 笔记打开时间 < 100ms
- 图谱渲染支持 1000+ 节点流畅交互
- 支持 10,000+ 笔记规模

### 5.2 隐私和安全
- 所有数据本地存储（Vault 在本地）
- API 调用使用加密传输
- 支持自部署 LLM（完全离线）
- 笔记内容不自动上传第三方

### 5.3 可扩展性
- 支持多 Vault 管理
- 支持插件扩展
- 支持自定义笔记模板
- 支持 API 集成（Zotero、Readwise 等）

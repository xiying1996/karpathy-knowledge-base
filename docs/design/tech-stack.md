## 1. 核心组件选型矩阵

### 1.1 笔记载体对比

| 方案 | 优点 | 缺点 | 推荐场景 | 评分 |
|------|------|------|----------|------|
| **Obsidian** | 本地 Markdown、生态丰富、社区活跃 | 闭源、图谱功能弱 | Karpathy 风格首选 | ⭐⭐⭐⭐⭐ |
| Logseq | 开源、大纲优先、Git 友好 | 插件生态较少 | 开源爱好者 | ⭐⭐⭐⭐ |
| Notion | 协作强、UI 精美 | 非本地、数据在云 | 团队协作 | ⭐⭐⭐ |
| Roam Research | 双向链接先驱、图谱强 | 贵、订阅制、图谱功能弱 | 深度研究 | ⭐⭐⭐ |
| RemNote | 间隔复习原生支持 | 较新、生态小 | 学生党 | ⭐⭐⭐ |

**结论**：选择 **Obsidian** 作为笔记载体
- Karpathy 本人使用
- 本地 Markdown，数据完全可控
- 插件生态丰富（Dataview、Templater、BRAT）
- 图谱通过社区插件可增强

### 1.2 Embedding 模型对比

| 模型 | 维度 | 中文支持 | 速度 | 成本 | 推荐 |
|------|------|----------|------|------|------|
| **BAAI/bge-large-zh-v1.5** | 1024 | ⭐⭐⭐⭐⭐ | 中 | 免费 | MVP 首选 |
| text-embedding-3-small (OpenAI) | 1536 | ⭐⭐⭐ | 快 | $0.02/1M | 云端备选 |
| text-embedding-3-large (OpenAI) | 3072 | ⭐⭐⭐ | 快 | $0.13/1M | 高精度场景 |
| mxbai-embed-large-v1 | 1024 | ⭐⭐⭐ | 中 | 免费 | BGE 替代 |
| Ollama 本地 (nomic-embed-text) | 768 | ⭐⭐ | 慢 | 免费 | 完全离线 |

**结论**：选择 **BAAI/bge-large-zh-v1.5**
- 中文效果领先
- 开源免费
- 支持 Ollama 本地部署

### 1.3 LLM 模型对比

#### 云端方案

| 模型 | 上下文 | 中文 | 成本 | 适用场景 |
|------|--------|------|------|----------|
| **GPT-4o** | 128K | ⭐⭐⭐⭐ | $5/1M in | 通用问答，质量最高 |
| Claude 3.5 Sonnet | 200K | ⭐⭐⭐⭐ | $3/1M in | 长文本理解 |
| **MiniMax M2.7** | 1M | ⭐⭐⭐⭐⭐ | ¥1/1M | 中文首选，性价比高 |
| Kimi (Moonshot) | 128K | ⭐⭐⭐⭐⭐ | ¥12/1M | 长上下文 |

#### 本地方案

| 模型 | 参数量 | 量化 | 中文 | GPU 需求 | 推荐场景 |
|------|--------|------|------|----------|----------|
| **Llama3.1 70B** | 70B | Q4 | ⭐⭐⭐ | 48GB+ | 高质量本地 |
| **Qwen2.5 72B** | 72B | Q4 | ⭐⭐⭐⭐⭐ | 48GB+ | 中文本地首选 |
| Llama3.1 8B | 8B | Q8 | ⭐⭐ | 8GB+ | 轻量本地 |
| Phi-3-medium | 14B | Q4 | ⭐⭐ | 16GB+ | 中等硬件 |

**结论**：分场景选择
- **默认**：MiniMax M2.7（中文好 + 便宜）
- **完全离线**：Qwen2.5 72B + Ollama
- **备用**：GPT-4o（质量优先时）

### 1.4 向量数据库对比

| 数据库 | 部署 | 扩展性 | 中文 | 成本 | 推荐场景 |
|--------|------|--------|------|------|----------|
| **ChromaDB** | 本地/云 | 一般 | ✅ | 免费 | MVP 首选 |
| Milvus | 云原生 | ⭐⭐⭐⭐⭐ | ✅ | 付费托管 | 生产环境 |
| Qdrant | 高性能 | ⭐⭐⭐⭐ | ✅ | 免费自部署 | Rust 生态 |
| Pinecone | 托管 | ⭐⭐⭐⭐⭐ | ✅ | 按量付费 | 免运维 |
| FAISS | 本地 | 差 | ✅ | 免费 | 超大规模 |

**结论**：选择 **ChromaDB**
- 轻量易部署
- Python 原生
- MVP 阶段足够
- 后续可迁移到 Milvus

### 1.5 图数据库对比

| 方案 | 存储 | 可视化 | 复杂度 | 推荐场景 |
|------|------|--------|--------|----------|
| **NetworkX + D3.js** | JSON | D3 力导向图 | 低 | MVP 首选 |
| Neo4j | 图数据库 | Bloom | 高 | 生产环境 |
| Obsidian Graph API | 内置 | 内置 | 极低 | 仅 Obsidian |
| Mermaid | 文本 | 简单视图 | 低 | 简单关系 |

**结论**：选择 **NetworkX + D3.js**
- 与笔记索引一起维护
- D3.js 可做复杂可视化
- 后续可升级 Neo4j

### 1.6 后端框架对比

| 框架 | 性能 | Python 生态 | 学习曲线 | 推荐 |
|------|------|-------------|----------|------|
| **FastAPI** | ⭐⭐⭐⭐⭐ | ✅ | 低 | MVP 首选 |
| Flask | ⭐⭐⭐ | ✅ | 极低 | 轻量场景 |
| LangChain | ⭐⭐⭐ | ✅ | 中 | RAG 快速原型 |
| LangGraph | ⭐⭐⭐ | ✅ | 高 | 复杂 Agent |

**结论**：选择 **FastAPI**
- 异步高性能
- 自动 OpenAPI 文档
- Pydantic 类型安全

### 1.7 前端框架对比

| 框架 | 性能 | 生态 | 学习曲线 | 推荐场景 |
|------|------|------|----------|----------|
| **React + Vite** | ⭐⭐⭐⭐⭐ | ⭐⭐⭐⭐⭐ | 中 | 复杂 Web UI |
| Vue | ⭐⭐⭐⭐ | ⭐⭐⭐⭐ | 低 | 快速开发 |
| Svelte | ⭐⭐⭐⭐⭐ | ⭐⭐ | 低 | 轻量应用 |
| 纯 HTML + JS | - | - | - | 简单界面 |

**结论**：选择 **React + Vite**
- 成熟生态
- 与 Obsidian 插件开发保持一致（JS）
- D3.js 集成方便

## 2. 推荐技术栈组合

### 2.1 MVP 版本（快速验证）

```
笔记载体:     Obsidian (本地 Markdown Vault)
后端框架:     FastAPI (Python)
向量数据库:   ChromaDB (本地)
Embedding:   BAAI/bge-large-zh-v1.5
LLM:         MiniMax M2.7 (云端) / Ollama (本地可选)
图谱存储:     NetworkX (内存) + JSON (持久化)
图谱可视化:   D3.js (Web)
前端:         React + Vite + TailwindCSS
部署:         Docker Compose (单机器)
```

### 2.2 Pro 版本（生产可用）

```
笔记载体:     Obsidian + Obsidian Sync
后端框架:     FastAPI + LangChain (可选)
向量数据库:   Milvus (云端)
Embedding:   BGE-large-zh-v1.5 (云端推理)
LLM:         GPT-4o / Claude 3.5 (云端)
图数据库:     Neo4j Aura
图谱可视化:   Obsidian Graph View + 自建 Web
前端:         React + Vite + Ant Design / shadcn/ui
部署:         Kubernetes (多机器)
监控:         Prometheus + Grafana
```

### 2.3 完全离线版本（最高隐私）

```
笔记载体:     Obsidian (本地 Vault)
后端框架:     FastAPI
向量数据库:   ChromaDB (本地)
Embedding:   Ollama + BGE (本地 GPU)
LLM:         Qwen2.5 72B + Ollama
图谱:         NetworkX + D3.js
部署:         本地服务器 + 内网穿透
```

## 5. 技术风险评估

### 5.1 高风险项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Embedding 质量不稳定 | RAG 效果差 | A/B 测试多模型，选择最优 |
| 笔记格式不统一 | 索引困难 | 强制定义模板规范 |
| 图谱性能瓶颈 | 1000+ 节点卡顿 | D3.js + Canvas 优化 |

### 5.2 中风险项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| Obsidian 版本升级破坏兼容性 | 功能异常 | 锁定版本 + CI 测试 |
| LLM API 涨价/不可用 | 成本激增 | 备用模型 + 本地 Ollama |
| Vault 损坏 | 知识丢失 | Git 备份 + 多设备同步 |

### 5.3 低风险项

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| ChromaDB 数据丢失 | 索引重建 | 定期备份 + 增量重建脚本 |
| 多语言混合（RAG 效果差） | 搜索不准 | 分离中英文 Embedding |

## 7. 决策总结

| 组件 | 最终选择 | 关键理由 |
|------|----------|----------|
| 笔记载体 | **Obsidian** | Karpathy 使用，本地 Markdown |
| 后端框架 | **FastAPI** | 高性能，Python 生态 |
| 向量数据库 | **ChromaDB** | 轻量，MVP 首选 |
| Embedding | **BGE-large-zh** | 中文最优，开源免费 |
| LLM | **MiniMax M2.7** | 中文好，¥1/1M 超便宜 |
| 图可视化 | **D3.js** | 灵活，可定制 |
| 前端 | **React + Vite** | 成熟生态 |

**MVP 总成本**：约 ¥50/月（仅 LLM API）
**MVP 部署**：单 Docker Compose，适合个人使用
